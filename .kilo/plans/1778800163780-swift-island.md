# Plan: Hapus Peserta Wawancara (Administrator Feature)

## Objective
Add a new menu item and page for administrators to delete/remove interview participants (peserta wawancara) from active or scheduled interview sessions.

## Context
- Current system has interview management for OSIS & MPK in `app/wawancara/`
- Only administrators can perform sensitive operations
- Participants (antrian) are created via QR code scanning
- Currently no way to remove participants once added (except status changes)

## Implementation Steps

### 1. Sidebar Navigation (components/layout/Sidebar.tsx)
- Add `UserX` icon import from lucide-react
- Add new menu item in `toolLinks` under `if (role === 'administrator')` block:
  ```js
  { href: '/hapus-peserta', label: 'Hapus Peserta Wawancara', icon: UserX }
  ```
- Position: after the QR Code item, before Log Aktivitas for logical grouping

### 2. Main Page (app/hapus-peserta/page.tsx)
- Server component with `getServerUser('administrator')` authorization
- Renders `HapusPesertaClient` component passing user data
- Returns redirect to `/dashboard` if not administrator

### 3. Client Component (app/hapus-peserta/HapusPesertaClient.tsx)
Main UI with:
- **Filters section**:
  - Organization filter (osis/mpk/all)
  - Session status filter (SCHEDULED/ACTIVE only - sessions that can be modified)
  - Search by name or class
- **Participants table** showing:
  - No, Nama, Kelas, Ekskul, Sesi (date/time), Status Antrian, Validasi, Hasil (if any)
  - Delete action button (trash icon) for each row
- **Delete confirmation** via ConfirmDialog
- Disabled state: participants from SELESAI/DIBATALKAN sessions
- Visual indicators for locked sessions

### 4. API Endpoint for Deletion (app/api/wawancara/antrian/[id]/route.ts)
**New file** with DELETE handler:
- Check `isAdministrator(ctx.userRole)` - only admin can delete
- Find antrian by id with sesi included
- Validate: session must be SCHEDULED or ACTIVE (not SELESAI/DIBATALKAN)
- Validate: participant status should allow deletion (MENUNGGU, SELESAI_WAWANCARA - but not WAWANCARA since currently being interviewed)
- Perform deletion with transaction:
  1. Delete associated `hasil_wawancara` record if exists (cascade or explicit)
  2. Delete the antrian record
- Create log entry via `createLog()` with action DELETE, tabel 'antrian_wawancara'
- Clear cache (`getCache.clear()` for wawancara data)
- Return success response

### 5. Additional Considerations
- **Authorization**: Use existing `isAdministrator` from `@/lib/auth-shared`
- **Logging**: Use existing `createLog` from `@/lib/log` - need to import it
- **Cache**: Clear relevant caches in wawancara API to reflect changes
- **Icons**: Use `UserX` from lucide-react for menu and page
- **Confirmation message**: Clear warning about irreversible deletion and associated data loss
- **Feedback**: Toast notifications for success/error

## Data Model Reference
- `antrianWawancara` table: id, sesi_id, nama, kelas, nomor_antrian, status (MENUNGGU/WAWANCARA/SELESAI_WAWANCARA), hasil_wawancara relation
- `sesiWawancara` table: id, status (SCHEDULED/ACTIVE/SELESAI/DIBATALKAN)
- `hasil_wawancara` table: antrian_id, keterangan, hasil, persentase, catatan

## Success Criteria
- Administrator sees "Hapus Peserta Wawancara" menu in Tools section
- Page loads list of participants from all active/scheduled sessions
- Admin can filter by organization
- Admin can delete participants with confirmation
- Deletion logs appear in activity log
- Cache invalidates properly - changes reflected immediately
